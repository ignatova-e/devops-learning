# SQL. Базовые запросы

## Что такое SQL

SQL (Structured Query Language) — язык запросов к реляционным базам данных.

С его помощью можно:

- получать данные;
- фильтровать записи;
- сортировать результаты;
- объединять таблицы;
- создавать и изменять структуру базы данных.

В этом разделе рассматриваются только базовые запросы на выборку данных
(`SELECT`).

---

## SELECT

Оператор `SELECT` используется для получения данных из таблицы.

```sql
SELECT *
FROM person;
```

Если необходимо вывести только определенные столбцы (name и age):

```sql
SELECT
    name,
    age
FROM person;
```

---

## WHERE

`WHERE` позволяет отфильтровать строки по условию. В структуре запроса идет
после `FROM`

```sql
SELECT *
FROM person
WHERE age > 18;
```

Можно использовать несколько условий.

```sql
WHERE age > 18
  AND gender = 'female';
```

или

```sql
WHERE address = 'Moscow'
   OR address = 'Kazan';
```

---

## BETWEEN

Используется для проверки диапазона значений.

```sql
WHERE age BETWEEN 18 AND 25;
```

Для дат:

```sql
WHERE visit_date
BETWEEN '2022-01-01' AND '2022-01-10';
```

---

## ORDER BY

Позволяет отсортировать результат. В структуре запроса всегда ставится в конце
запроса.

По возрастанию:

```sql
ORDER BY age;
```

Чтобы явно указать возрастание можно писать так:

```sql
ORDER BY age ASC;
```

По убыванию:

```sql
ORDER BY age DESC;
```

По нескольким столбцам:

```sql
ORDER BY
    age,
    name;
```

---

## DISTINCT

Удаляет повторяющиеся строки. Ставится сразу после ключевого слова `SELECT`

```sql
SELECT DISTINCT address
FROM person;
```

---

## CASE

Позволяет вернуть разные значения в зависимости от условия.

```sql
CASE
    WHEN age < 18 THEN 'child'
    WHEN age < 60 THEN 'adult'
    ELSE 'senior'
END
```

Используется внутри `SELECT`.

---

## FORMAT

Позволяет объединить несколько значений в одну строку.

```sql
SELECT
    format(
        '%s (age:%s)',
        name, age
    )
FROM person;
```

---

## Подзапросы

Подзапрос — это запрос внутри другого запроса.

Например:

```sql
SELECT (
    SELECT name
    FROM person
    WHERE id = person_order.person_id
    )
FROM person_order;
```

Подзапросы могут использоваться:

- в SELECT;
- в FROM;
- в WHERE.

---

## Проверка существования данных

### EXISTS

Возвращает TRUE, если подзапрос нашёл хотя бы одну строку.

```sql
WHERE EXISTS (...)
```

---

### NOT EXISTS

Используется для поиска отсутствующих записей.

```sql
WHERE NOT EXISTS (...)
```

---

## Операции над множествами

SQL позволяет работать с результатами нескольких запросов.

### UNION

Объединяет результаты и удаляет дубликаты.

### UNION ALL

Объединяет результаты и сохраняет дубликаты.

### INTERSECT

Возвращает строки, которые есть в обоих запросах.

### EXCEPT

Возвращает строки первого запроса, отсутствующие во втором.

### EXCEPT ALL

Работает как EXCEPT, но сохраняет количество повторений.

---

## generate_series()

Функция PostgreSQL для генерации последовательностей.

Например:

```sql
generate_series(
    '2022-01-01',
    '2022-01-10',
    interval '1 day'
)
```

Позволяет получить все даты в диапазоне.

Часто используется вместе с LEFT JOIN для поиска пропущенных дат.

---

## Объединение таблиц

Для получения информации из нескольких таблиц используются JOIN.

Например:

```sql
person
  │
  ├──────── person_order
  │
  └──────── person_visits
                  │
                  └──────── pizzeria
```

Я использовала:

- JOIN
- LEFT JOIN
- NATURAL JOIN

---

### JOIN

Возвращает строки, найденные в обеих таблицах.

```sql
SELECT *
FROM person
JOIN person_order
ON person.id = person_order.person_id;
```

---

### LEFT JOIN

Возвращает все строки из левой таблицы.

Если совпадений нет — значения справа будут NULL.

Часто используется вместе с

```sql
WHERE right_table.column IS NULL
```

для поиска отсутствующих записей.

---

### NATURAL JOIN

Автоматически соединяет таблицы по одинаковым названиям столбцов.

Использовать нужно осторожно.

Если одинаковые столбцы имеют другое назначение, результат может оказаться
неправильным.

### FULL JOIN

`FULL JOIN` возвращает:

- все строки из левой таблицы;
- все строки из правой таблицы;
- совпавшие строки объединяются.

Если для строки не найдено соответствие во второй таблице, недостающие значения
будут заполнены `NULL`.

```sql
SELECT *
FROM person
FULL JOIN person_visits
    ON person.id = person_visits.person_id;
```

Часто используется, когда необходимо получить полный набор данных из обеих
таблиц.

### SELF JOIN

Иногда одну и ту же таблицу необходимо соединить саму с собой.

Для этого используются псевдонимы.

```sql
SELECT
    p1.name,
    p2.name
FROM person AS p1
JOIN person AS p2
    ON p1.address = p2.address;
```

Такой прием называется **самосоединением (SELF JOIN)**.

Часто используется для поиска:

- людей из одного города;
- сотрудников одного отдела;
- одинаковых записей;
- пар объектов.

---

## COALESCE

Функция `COALESCE()` позволяет заменить `NULL` на указанное значение.

Например:

```sql
SELECT
    COALESCE(name, '-') AS name
FROM person;
```

Если `name` равно `NULL`, будет выведено `'-'`.

Можно использовать несколько аргументов.

```sql
COALESCE(value1, value2, value3)
```

Функция вернет первое значение, которое не является `NULL`.

---

## CTE (Common Table Expression)

CTE позволяет сохранить результат запроса под временным именем и использовать
его как обычную таблицу.

Создается с помощью оператора `WITH`.

```sql
WITH dates AS (
    SELECT generated_date::date AS missing_date
    FROM generate_series(
        '2022-01-01'::date,
        '2022-01-10'::date,
        INTERVAL '1 day'
    ) AS generated_dates(generated_date)
)
SELECT *
FROM dates;
```

Преимущества:

- делает запрос более читаемым;
- позволяет избежать повторения одинаковых подзапросов;
- удобно разбивать сложный запрос на этапы.

---

## GROUP BY

Используется для объединения строк в группы.

Чаще всего применяется вместе с агрегатными функциями.

```sql
SELECT
    address,
    COUNT(*)
FROM person
GROUP BY address;
```

После выполнения каждая группа превращается в одну строку.

---

## HAVING

Фильтрует уже сформированные группы.

В отличие от `WHERE`, который работает **до** группировки, `HAVING` применяется
**после** `GROUP BY`.

```sql
SELECT
    address,
    COUNT(*)
FROM person
GROUP BY address
HAVING COUNT(*) > 1;
```

---

## COUNT

`COUNT` — агрегатная функция для подсчета строк.

Количество всех строк:

```sql
COUNT(*)
```

Количество непустых значений:

```sql
COUNT(column)
```

Количество уникальных значений:

```sql
COUNT(DISTINCT column)
```

---

## DISTINCT внутри агрегатных функций

`DISTINCT` можно использовать не только после `SELECT`, но и внутри агрегатных
функций.

Например:

```sql
COUNT(DISTINCT pizza_name)
```

Будут подсчитаны только уникальные значения.

Это удобно, если один и тот же объект встречается несколько раз.

---

## Псевдонимы (AS)

Чтобы сделать запрос более читаемым, таблицам и столбцам можно задавать
временные имена.

Для столбцов:

```sql
SELECT
    name AS person_name
FROM person;
```

Для таблиц:

```sql
FROM person AS p
```

После этого к столбцам можно обращаться через псевдоним.

```sql
p.name
```

Особенно важно использовать псевдонимы при работе с несколькими таблицами или
при SELF JOIN.

---

## Логический порядок выполнения SELECT

Если запрос содержит `WITH` (CTE), сначала выполняются все CTE.

После этого начинается выполнение основного запроса.

Хотя запрос пишется так:

```sql
WITH ...
SELECT ...
FROM ...
WHERE ...
GROUP BY ...
HAVING ...
ORDER BY ...
```

Логически SQL выполняет его примерно в таком порядке:

1. `WITH` (если используется)
2. `FROM`
3. `JOIN`
4. `ON`
5. `WHERE`
6. `GROUP BY`
7. `HAVING`
8. `SELECT`
9. `DISTINCT`
10. `ORDER BY`

Понимание этого порядка помогает объяснить, почему:

- в `WHERE` нельзя использовать агрегатные функции;
- `HAVING` работает только после `GROUP BY`;
- сортировка выполняется самой последней;
- `CTE` сначала формирует временный набор данных, который затем используется в
  основном запросе.

## Что стоит запомнить

После прохождения этой темы вы должны уметь:

- выбирать данные с помощью SELECT;
- фильтровать строки через WHERE;
- сортировать результаты;
- использовать BETWEEN;
- объединять таблицы через JOIN;
- писать простые подзапросы;
- понимать разницу между UNION, INTERSECT и EXCEPT;
- использовать generate_series() для работы с диапазонами дат.
